<summary><b>📊 Faculty Workload Management System (FWMS) — Academic Analytics Platform · Python · FastAPI · PostgreSQL · React 18</b></summary>

<br/>

> Institutional-grade academic workload allocation engine with 4-tier RBAC, automated analytics exports, and real-time deviation tracking across 50+ faculty and 150+ courses.

| Dimension | Details |
|-----------|---------|
| **Stack** | Python · FastAPI · PostgreSQL · React 18 · TypeScript · Google OAuth 2.0 |
| **Scale** | 22-table schema · 50+ faculty · 4 academic programs · 150+ courses · 19,200+ offerings |
| **API Surface** | 30+ REST endpoints · Pydantic validation · SQLAlchemy ORM |
| **Security** | 4-tier RBAC (Faculty / Coordinator / HOD / Admin) · Full audit logging |
| **Analytics** | 3-sheet Excel exports · 15 metrics per faculty · PDF summaries · Power BI KPIs |
| **Impact** | Delivered for institutional adoption · Replaced fully manual allocation process |

- Implemented a **5-pass progressive allocation algorithm** enforcing 9 business rules — guaranteeing equitable course distribution across departments while respecting faculty constraints
- Designed complex **SQL analytics** using LEFT JOINs, COALESCE, GROUP BY, and subqueries to compute per-faculty TCH totals, deviation vs. 18-hour norm, and classify workload as Overloaded / Balanced / Underloaded
- Engineered **department-level KPIs** — allocation success rate, overload percentage, and workload distribution curves — surfaced through automated Excel exports and Power BI dashboards
- Built a complete **audit logging system** with timestamped role-action records and a Google OAuth 2.0 authentication layer supporting 4 user tiers
